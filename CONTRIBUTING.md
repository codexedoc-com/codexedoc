Thank you for your interest in contributing!

## Before You Start

Before beginning work:

- Check whether an issue already exists.
- If you're working on a large feature, start a discussion first.
- If you'd like to work on an existing issue, leave a comment so others know it's being worked on.
---

## Development Setup

Follow the installation instructions in the project's **README.md** to get a local development environment running before contributing.

---

## Branch Naming

```
user/<github-username>/<issue-number>
```

Example:

```
user/glenniii-dev/42
```

---

## Commit Messages

```
[issue-number] Short description
```

Example:

```
[42] Add authentication middleware
```

---

## Pull Requests

Please:

* Link related issues
* Describe what changed
* Explain why the change is needed
* Ensure all checks pass
* Request review before merging

---

## Code Style

* TypeScript only
* Use strict typing
* Prefer reusable components
* Avoid `any` where possible
* Follow existing project conventions

Run before submitting:

```bash
npm run lint
```

---

## Communication

Development discussions take place on the project's Discord server.

Use:

- GitHub Issues for bug reports and feature requests.
- Discord for questions, design discussions, and contributor coordination.

---

## Documentation

Update documentation whenever behavior changes.

This includes README updates when installation or usage changes.
